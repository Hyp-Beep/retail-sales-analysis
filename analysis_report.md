# 零售订单数据分析项目 · 分析报告
> 基于 70,690 条真实零售设备订单数据的全流程分析
> 项目地址：https://github.com/Hyp-Beep/retail-sales-analysis

---

# 一、项目背景与目标
本项目以自动售货机/零售设备的真实订单数据为基础，通过 **Python 数据清洗 + SQL 多维分析 + 可视化展示**，为零售点位运营、商品补货策略、设备管理提供数据依据。

分析目标包括：
1. 统计整体订单规模与销售额
2. 识别各点位表现差异
3. 分析商品品类结构与热销商品
4. 提供可落地的运营优化建议

---

# 二、数据概况
- 总订单量：**70,690 条**
- 总销售额：**285,898.50 元**
- 时间范围：2017-01（单月数据）
- 数据覆盖：订单号、商品、金额、点位、分类、支付时间等字段

数据质量：
- 无缺失值
- 无异常字段
- 所有订单为有效交易（状态统一为“已出货未退款”）

---

# 三、技术流程说明
本项目采用标准数据分析工程流程：

1. **数据获取**：从 CSV 读取原始订单与商品分类数据
2. **数据清洗**：Pandas 完成格式转换、去重、字段关联
3. **数据存储**：写入 SQL Server，形成可持久化数据表
4. **多维分析**：通过 SQL 完成点位、商品、品类、销售额分析
5. **可视化**：Matplotlib 生成业务图表 → 直观呈现数据结论
6. **结果输出**：形成运营建议与项目文档

---

# 四、核心分析结果
下面是项目中最关键的业务洞察（面试官/老师最看重的部分）：

## 1. 整体运营表现
- 总订单：70,690 单
- 总销售额：285,898.50 元
- 平均单客价：约 4.04 元
- 订单状态稳定，无大量退款/异常单

→ 说明运营状态良好，设备运转正常。

---

## 2. 点位表现分析
根据 SQL 统计，各点位销量与销售额差异明显：

| 点位 | 销量 | 销售额 | 占比 |
|-----|------|---------|-------|
| E | 23,507 | 95,397.20 | **33.4%** |
| C | 14,499 | 61,320.90 | 21.4% |
| B | 13,481 | 53,601.50 | 18.8% |
| A | 10,489 | 42,389.10 | 14.8% |
| D | 8,714 | 33,189.80 | 11.6% |

### 关键结论
- **E 点位是绝对核心盈利点**，贡献三分之一销售额。
- D 点位表现较弱，需要调整商品结构或加强运营。

→ 运营建议：对 E 点位提升补货频率，保证热门商品不断货。

---

## 3. 商品品类结构分析
SQL 大类统计结果：

- 饮料类销量：**占比约 55%**
- 非饮料类销量：**占比约 45%**

结论：
- 饮料与非饮料占比均衡，符合自动售货机真实消费结构。
- 品类设置合理，可继续维持稳定商品组合。

---

## 4. 热销商品 TOP10
SQL 结果显示：
- 高频商品集中在：
  - 可口可乐系列
  - 好丽友派
  - 双汇热狗肠
  - 营养快线等

→ 这些商品是点位的“主力营收来源”，应该优先备货。

# 六、业务优化建议
结合 SQL 分析结果，给出可直接落地的运营方案：

### 1. 点位运营策略
- 重点保障 E 点位库存，防止热门商品断货
- 对 D 点位试点商品替换，提高动销率
- 对 A/B/C 点位维持稳定补货节奏

### 2. 商品备货策略
- 优先备货 TOP10 热销商品
- 饮料类可增加 10–20% 陈列空间
- 低频商品可逐步淘汰，为高毛利单品腾位置

### 3. 设备管理
- 监控高销量点位的设备状态，避免故障
- 根据时段分布优化补货时间（例如早高峰/晚高峰）
