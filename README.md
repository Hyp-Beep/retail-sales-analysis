# 零售订单数据分析项目
📊 基于70690条零售设备订单数据的全流程分析

## 项目简介
本项目以真实零售设备订单数据为基础，构建了一套完整的数据分析流程，从数据清洗、多表关联、SQL持久化存储，到业务指标分析与可视化，为零售点位运营、商品补货策略提供数据支撑。

## 技术栈
- **数据处理**: Python 3.x / Pandas
- **数据存储**: SQL Server
- **数据可视化**: Matplotlib
- **版本控制**: Git & GitHub

## 文件说明
- `data1.csv`：原始订单主数据
- `data2.csv`：商品分类数据
- `main.py`：项目主程序（数据处理+入库+可视化）
- `load.py`：数据加载与预处理模块
- `plot.py`：可视化绘图模块
- `db_tools.py`：SQL Server 数据入库脚本
- `retail_analysis.sql`：核心业务分析 SQL 脚本

## 核心分析结论
1.  **整体运营**: 总订单量70690条，总销售额285898.50元，订单均为有效交易，运营状态良好。
2.  **点位表现**: E点位为核心盈利点，贡献约33%销售额；D点位销量与营收均偏低，需优化商品结构。
3.  **商品结构**: 饮料与非饮料品类销量均衡，符合自动售货机消费场景，商品结构健康。

## 项目亮点
- ✅ 完整的数据处理流程，覆盖从原始数据到业务结论的全链路
- ✅ 结合SQL实现数据持久化与多维度聚合查询，而非仅依赖内存处理
- ✅ 分析结果可直接落地为运营建议，体现数据驱动业务的价值
